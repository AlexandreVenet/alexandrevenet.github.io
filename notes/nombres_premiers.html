Nombres premiers

Exercice sur les nombres premiers, en Php.

Objectif : écrire une fonction qui retourne un tableau contenant les x premiers nombres premiers jusqu'à la valeur n passée en paramètre.

Un nombre premier est un nombre entier, divisible par 1 et par lui-même et c'est tout. Donc, s'il est divisible par autre chose, il n'est pas premier.

Utiliser cette définition n'est pas suffisant pour discriminer les nombres entiers car "divisible par 1 et par lui-même" vaut pour tous les nombres. Donc, préférer plutôt procéder par élimination de ce qui n'est pas premier.

Pour savoir si un nombre n'est pas premier, il faut qu'il puisse être divisible par un autre nombre. Or, "être divisible par un autre nombre" signifie que le résultat de la division est un entier. Donc, un nombre est premier si la division avec autre chose donne un résultat non entier, c'est-à-dire avec un reste, des valeurs décimales.

Or, "division avec autre chose" n'est pas suffisant car une seule division ne permet pas à elle seule de savoir si le nombre est premier. Donc, il faut tester plusieurs divisions et que tous les résultats aient été avec reste, pour dire que tel nombre est premier. Or, quels nombres utiliser ?

  • On sait que diviser un nombre par un autre qui lui est supérieur donne des valeurs décimales, donc des résultats avec reste. Ce n'est pas éclairant car dans ce cas, tout nombre est premier et en plus on tend vers l'infini (fonction qui ne s'arrêterait jamais). Donc, on va éliminer la division par nombre supérieur.
  • Diviser par 0 n'est pas disciminant non plus car on admet alors soit que la division est impossible (fin de l'affaire), soit que son résultat est l'infini (fin de l'affaire car impossible de savoir si c'est premier ou non). Donc, on élimine aussi 0.
  • Diviser par 1 : on pourrait garder mais il y a 2 raisons d'élimination. (1) Tout nombre divisé par 1 donne lui-même, ce qui n'est pas discriminant. (2) 1 divisé par 1, c'est certes "diviser par lui-même", mais ce n'est pas "diviser par 1" (ou inversement) en tant que 1 serait un autre nombre que "lui-même" ; ainsi la formule être "divisible par 1 et par lui-même" n'est pas satisfaite. Par conséquent, la règle du nombre premier dit plus précisément : être divisible par 1 en tant que c'est un autre nombre et par lui-même. On peut se demander si cela signifie qu'un nombre n'est pas en identité avec sa valeur, ce qui serait étonnant.
  • Diviser à partir de 2 semble discriminant (un nombre divisé par 2 dont le résultat est sans reste est pair, donc non premier).

Pour savoir si une division a du reste, on dispose de l'opération modulo (`%`).


<?php
function premiers(int $n){
	$premiers = []; // le tableau de valeurs
	for($i = 2; $i < $n; $i++){// de 2 au nombre fourni
		$premier = true; // hypothèse
		for($j = 2; $j < $i; $j++){// on teste $i avec tous ses précédents
			$reste = $i % $j; // savoir s'il y a un reste de $i/$j
			if($reste == 0){ // si pas de reste, alors on a un nombre entier
				$premier = false; // invalidation de l'hypothèse
				break; // arrêter la boucle
			}
		}
		if($premier){ // après test, si hypothèse non invalidée...
			$premiers[] = $i; // ... alors, on a un premier, on le conserve dans le tableau
		}
	}
	return $premiers; // renvoyer
}
print_r(premiers(100));
